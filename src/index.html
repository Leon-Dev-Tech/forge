WeChat Image Long-press Solution

This page demonstrates why images inside Shadow DOM cannot be long-pressed in WeChat and provides solutions.

🌟 Recommended Architecture: MFE Event -> Host Bridge

Use Case: Perfect for strict micro-frontend environments. The MFE component emits a WECOM_GHOST_REGISTER event and never touches the global document. The Host application (Base) listens to the event and safely mounts the ghost image to bypass WeCom restrictions.

🆕 New: Base64 + WeixinJSBridge Solution

Use Case: When images require authentication (cookies) and imagePreview fails to fetch them. This solution fetches the image manually and converts it to Base64 before previewing.

🔵 Experimental: Document Ghost Image (WeCom Hack)

Use Case: When WeixinJSBridge.invoke('imagePreview') fails in Enterprise WeChat (WeCom). This mounts a transparent <img> over the component in the main document's body via ownerDocument, allowing native hit-testing to work.

✅ Recommended: MFE Safe Bridge (globalThis)

Status: Working. Bypasses Shadow DOM and window restrictions. Click the image to trigger native WeChat preview.

❌ The Problem: Standard Shadow DOM

Status: Broken. WeChat cannot "see" this image for long-press menus because it is inside a Shadow Root.

❌ Deep Problem: Nested Shadow DOM

Status: Broken. Even if the child is Light DOM, a Shadow DOM ancestor will block WeChat's recognition.

ℹ️ Control: Light DOM

Status: Working. Standard Stencil component with shadow: false.

ℹ️ Alternative: Pure HTML Link

Status: Working (Less polished). Opens image in a new tab. MFE safe but changes page context.

ℹ️ Baseline: Native HTML Image

Status: Working. Just a standard <img> tag.
